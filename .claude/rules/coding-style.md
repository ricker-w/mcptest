# Coding Style — 汎用コーディング規約

言語に依存しない品質基準。全プロジェクト・全エージェントが遵守すること。

---

## 関数・メソッド

| # | ルール |
|---|--------|
| CS-1 | 1関数1責務。関数は **50行以下** に保つ |
| CS-2 | 関数名は **動詞 + 名詞** で何をするか明示する（`getUserById`, `validateInput`） |
| CS-3 | 引数は **4つ以下**。超える場合はオブジェクト/構造体でまとめる |
| CS-4 | 早期リターンでネストを浅くする。**ネストは4段以下** |

## ファイル

| # | ルール |
|---|--------|
| CF-1 | 1ファイル1モジュール。ファイルは **800行以下** に保つ |
| CF-2 | 超過したら責務で分割する |
| CF-3 | import/require は ファイル先頭にまとめる（stdlib → 外部 → ローカル） |

## 命名

| # | ルール |
|---|--------|
| CN-1 | 変数・関数名は **意味が分かる名前** にする。1-2文字の略語は禁止（ループ変数 `i`, `j` を除く） |
| CN-2 | 定数は **UPPER_SNAKE_CASE** |
| CN-3 | boolean は `is`, `has`, `can`, `should` で始める |

## イミュータビリティ

| # | ルール |
|---|--------|
| CI-1 | **可能な限り不変** にする。新しいオブジェクトを生成し、既存を変更しない |
| CI-2 | 変更可能な変数には理由を明記する |

## エラーハンドリング

| # | ルール |
|---|--------|
| CE-1 | エラーを **握りつぶさない**。catch した場合はログ出力 or 再throw |
| CE-2 | ユーザー向けメッセージと開発者向けメッセージを分離する |
| CE-3 | エラーにはコンテキスト（何を、どこで、なぜ失敗したか）を含める |

## 定数・マジックナンバー

| # | ルール |
|---|--------|
| CM-1 | ハードコーディングされた値は **定数に切り出す** |
| CM-2 | 設定値は環境変数または設定ファイルで管理する |

## コメント

| # | ルール |
|---|--------|
| CC-1 | **なぜ（Why）** をコメントする。何（What）はコード自体で表現する |
| CC-2 | 自明なコメント（`// increment i`）は書かない |
| CC-3 | TODO / FIXME は Issue 番号を添える |
