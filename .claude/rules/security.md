# Security — 汎用セキュリティルール

全プロジェクト・全エージェントが遵守するセキュリティ基準。

---

## 入力バリデーション

| # | ルール |
|---|--------|
| SI-1 | **外部入力は全て信頼しない**。ユーザー入力、API レスポンス、ファイル内容、環境変数 |
| SI-2 | バリデーションは **システム境界で行う**。内部コード間では不要 |
| SI-3 | ホワイトリスト方式を優先する（許可するものを定義。禁止リストではない） |

## インジェクション防止

| # | ルール |
|---|--------|
| SJ-1 | SQL はパラメータ化クエリ / プリペアドステートメントを使う。文字列結合で SQL を組み立てない |
| SJ-2 | HTML 出力はエスケープする（XSS 防止） |
| SJ-3 | シェルコマンドにユーザー入力を直接渡さない |
| SJ-4 | ファイルパスは正規化してからアクセスする（パストラバーサル防止） |

## 認証・認可

| # | ルール |
|---|--------|
| SA-1 | 認証と認可を分離する。認証（誰か）→ 認可（何ができるか） |
| SA-2 | セッショントークンは httpOnly + secure + sameSite で管理 |
| SA-3 | API キーはヘッダーで渡す。URL パラメータに含めない |

## シークレット管理

| # | ルール |
|---|--------|
| SS-1 | シークレットをコードにハードコーディングしない |
| SS-2 | `.env` ファイルは `.gitignore` に含める |
| SS-3 | ログにシークレットを出力しない |
| SS-4 | `.claude/` ディレクトリは Git にコミットしない |

## 依存関係

| # | ルール |
|---|--------|
| SD-1 | 依存パッケージは定期的に脆弱性スキャンする |
| SD-2 | ロックファイル（package-lock.json, Cargo.lock 等）をコミットする |
| SD-3 | 不要な依存は削除する（攻撃面の縮小） |

## HTTP ヘッダー

| # | ルール |
|---|--------|
| SH-1 | Content-Security-Policy を設定する |
| SH-2 | X-Content-Type-Options: nosniff を設定する |
| SH-3 | X-Frame-Options で clickjacking を防止する |
| SH-4 | CORS は必要最小限のオリジンのみ許可する |

## ファイル操作

| # | ルール |
|---|--------|
| SF-1 | ファイルパスを組み立てる際は **パストラバーサルを検証** する |
| SF-2 | アップロードファイルはサイズ・種別を制限する |
| SF-3 | テンポラリファイルは使用後に削除する |
