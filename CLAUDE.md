# TeamUp — CLAUDE.md

## Project Overview

This is a **template repository** for AI-driven new business ideation using Claude Teams. It is not a software product itself — it is a starting point that gets cloned to bootstrap new project repositories.

When cloned, it provides:
- Specialist AI agent prompts (CEO, Market Researcher, Tech Lead, etc.)
- A 4-phase orchestrator template (Research → Tech Design → Product Def → Spec Writing)
- Shared rules enforced across all agents
- Hook scripts for cost tracking and safety

## Repository Purpose

**Do not implement application code here.** This repository contains only:
- Agent prompts (`.claude/agents/`)
- Orchestration blueprints (`.claude/orchestrators/`)
- Shared rules (`.claude/rules/`)
- Reusable skills (`.claude/skills/`)
- Context overlays (`.claude/contexts/`)

Generated output (`docs/`, `logs/`) is created at runtime in the **cloned project repository**, not here.

## Key Files

| Path | Description |
|------|-------------|
| `.claude/orchestrators/product-ideation-template.md` | Main orchestrator — customize `{...}` fields per project |
| `.claude/agents/ceo.md` | Team Leader SA prompt (CEO persona) |
| `.claude/rules/orchestration.md` | Full orchestration protocol — read this first |
| `.claude/hooks/hooks.json` | Hook configuration for Bash pre-check and cost tracking |

## How to Launch

1. Clone this repo as a new project repository
2. Edit `.claude/orchestrators/product-ideation-template.md` — fill in all `{...}` placeholders
3. Tell the CEO (Team Leader) to read the orchestrator and start Phase 1

```
Read .claude/orchestrators/product-ideation-template.md and begin Phase 1.
```

## Rules Reference

All `.claude/rules/*.md` files are enforced across all agents:

- `orchestration.md` — 3-tier architecture, communication protocol, quality gates
- `tdd-practices.md` — TDD cycle, test pyramid, coverage targets
- `branching-strategy.md` — Git flow, PR process, commit conventions
- `coding-style.md` — Code quality standards (language-agnostic)
- `security.md` — Security baseline (input validation, injection prevention, secrets)

## Quality Gates

Agents must pass all quality gates before completing a task:

| Gate | Check |
|------|-------|
| QG-1 | All tests pass |
| QG-2 | Lint passes with zero errors |
| QG-3 | E2E tests pass (if applicable) |
| QG-4 | All DoD items checked |
| QG-5 | Code quality and security review |

## Agents Directory

Each agent in `.claude/agents/` follows the CRISP format:
- **C**ontext — situation and preconditions
- **R**ole — persona, expertise, what the agent does and doesn't do
- **I**ntent — goal and success criteria
- **S**teps — execution procedure and output format
- **P**roof — verification checklist

## Prohibited Actions

- Direct push to `main` or `develop`
- Committing `.env` or `.claude/` directories
- Using `git --no-verify` (blocked by hook)
- Teammate-to-Teammate direct communication
- Teammates reading orchestrator or rules files directly
